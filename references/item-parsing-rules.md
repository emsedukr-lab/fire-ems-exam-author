# Item Parsing Rules

## 질문 시작 패턴

우선 탐지할 패턴:
- `1.`
- `1)`
- `01.`
- `1번`

질문 시작 줄은 보통 다음 성질을 가진다:
- 숫자로 시작
- 뒤에 6자 이상 본문이 뒤따름
- 정답표처럼 짧은 `1. ②` 형식이 아님

## 선택지 패턴

지원 기본 패턴:
- `① ② ③ ④ ⑤`
- `1) 2) 3) 4)`
- `A. B. C. D.`
- `A) B) C) D)`

## 문항 후보 인정 기준

- 질문 줄 또는 질문 블록이 존재
- 선택지가 최소 2개 이상
- stem이 비어 있지 않음

아래는 자동 승인 금지:
- 선택지 2개 미만
- 질문 번호는 있으나 stem이 거의 없음
- 정답표와 섞여 번호만 반복되는 텍스트

## 정답표 탐지 규칙

정답표 가능성이 높은 패턴:
- `1. ②`
- `1) 2`
- `1-A`
- `1번 ③`

정답표 문서는 대체로:
- 짧은 줄 위주
- 숫자-정답 패턴 밀도 높음
- 질문 stem과 긴 선택지 설명이 거의 없음

## OCR 품질 위험 신호

- `■`, `�` 같은 대체 문자 다수
- 한글이 의미 없는 영숫자로 깨짐
- 선택지 라벨이 순서대로 나오지 않음
- 질문과 선택지가 한 줄에 비정상적으로 합쳐짐

이 경우 `quality_flags`에 반영하고 기본적으로 `needs_review`로 보낸다.
